# Progress Log

## Learnings
- Prisma 7.x: datasource URL goes in `prisma.config.ts` (not `schema.prisma`). The `url` field in schema.prisma is rejected.
- `npx prisma validate` is the correct command to validate schema syntax without a DB connection.
- `npx prisma generate` works without a live DB (generates TS client from schema).
- `npx tsc --noEmit` validates TypeScript without emitting files.
- When no PostgreSQL is available locally, migration SQL is written manually and validated via schema validation + typecheck.
- Project stack: Next.js 15 (App Router), TypeScript, Prisma 7, PostgreSQL. Code lives in `app/` subdirectory.
- Prisma 7 PrismaClient constructor requires either `accelerateUrl` or `adapter` â€” TS type rejects plain `{ log: [...] }` options. Workaround: cast constructor to `any` and type the result as `PrismaClient`.

## Iteration 10 â€” US-014: Asset lifecycle transition API with soft enforcement
- What was done: Created `src/app/api/assets/[id]/transition/route.ts` with `POST /api/assets/:id/transition`. Logic: (1) Load asset with current stage + family; (2) Validate target stage exists; (3) Query `lifecycle_transitions` for an active transition matching from_stage + to_stage + (family-specific OR applies-to-all); (4) If no transition found â†’ 422; (5) Check `requiredDocumentTypes` (look up by name in document_types, check documents table for asset attachment) and `requiredEvents` (look up by name in event_types, check events table for asset); (6) If warnings and no `override_warnings` â†’ return 200 with warnings array and `transition_blocked: false`; (7) If proceeding â†’ atomically update `asset.current_lifecycle_stage_id` + create `lifecycle_stage_changed` governance event with metadata including `from_stage_id/name`, `to_stage_id/name`, and `warnings_overridden`/`justification` if applicable. Ran `tsc --noEmit` â†’ clean.
- Files affected: `app/src/app/api/assets/[id]/transition/route.ts`
- Learnings for next cycles:
  - Patterns: Use `prisma.$transaction([...])` with spread for conditional extra creates â€” spread an empty array when the governance event type is not found (graceful degradation).
  - Patterns: Required document types and required events in `lifecycle_transitions` are stored as arrays of names (strings) in JSONB â€” look them up by name at runtime.
  - Patterns: Soft enforcement: warnings never block transitions; `transition_blocked` is always `false`.
