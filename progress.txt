# Progress Log

## Learnings
- Prisma 7.x: datasource URL goes in `prisma.config.ts` (not `schema.prisma`). The `url` field in schema.prisma is rejected.
- `npx prisma validate` is the correct command to validate schema syntax without a DB connection.
- `npx prisma generate` works without a live DB (generates TS client from schema).
- `npx tsc --noEmit` validates TypeScript without emitting files.
- When no PostgreSQL is available locally, migration SQL is written manually and validated via schema validation + typecheck.
- Project stack: Next.js 15 (App Router), TypeScript, Prisma 7, PostgreSQL. Code lives in `app/` subdirectory.
- Prisma 7 PrismaClient constructor requires either `accelerateUrl` or `adapter` — TS type rejects plain `{ log: [...] }` options. Workaround: cast constructor to `any` and type the result as `PrismaClient`.

## Iteration 10 — US-014: Asset lifecycle transition API with soft enforcement
- What was done: Created `src/app/api/assets/[id]/transition/route.ts` with `POST /api/assets/:id/transition`. Logic: (1) Load asset with current stage + family; (2) Validate target stage exists; (3) Query `lifecycle_transitions` for an active transition matching from_stage + to_stage + (family-specific OR applies-to-all); (4) If no transition found → 422; (5) Check `requiredDocumentTypes` (look up by name in document_types, check documents table for asset attachment) and `requiredEvents` (look up by name in event_types, check events table for asset); (6) If warnings and no `override_warnings` → return 200 with warnings array and `transition_blocked: false`; (7) If proceeding → atomically update `asset.current_lifecycle_stage_id` + create `lifecycle_stage_changed` governance event with metadata including `from_stage_id/name`, `to_stage_id/name`, and `warnings_overridden`/`justification` if applicable. Ran `tsc --noEmit` → clean.
- Files affected: `app/src/app/api/assets/[id]/transition/route.ts`
- Learnings for next cycles:
  - Patterns: Use `prisma.$transaction([...])` with spread for conditional extra creates — spread an empty array when the governance event type is not found (graceful degradation).
  - Patterns: Required document types and required events in `lifecycle_transitions` are stored as arrays of names (strings) in JSONB — look them up by name at runtime.
  - Patterns: Soft enforcement: warnings never block transitions; `transition_blocked` is always `false`.

## Iteration 14 — US-018: User authentication schema and API
- What was done: Added `UserRole` enum and `User` model to `prisma/schema.prisma` with all required columns (id, email, name, password_hash, role, responsible_body_id FK, is_active, last_login_at, created_at, updated_at). Added `users` reverse relation to `ResponsibleBody`. Created migration SQL at `prisma/migrations/0018_users/migration.sql` with CREATE TYPE user_role, CREATE TABLE users, and ALTER TABLE FK constraint resolutions for previously-stubbed FKs in events (recorded_by_user_id) and documents (uploaded_by_user_id, verified_by_user_id). Installed `bcryptjs` and `jsonwebtoken` plus their @types packages. Created `src/lib/jwt.ts` with signAccessToken (15m TTL), signRefreshToken (7d TTL), verifyToken helpers. Created `src/lib/auth.ts` with getUserFromRequest helper that extracts and validates Bearer token from Authorization header. Created 4 auth route handlers: `POST /api/auth/register` (admin-only, creates user with bcrypt 12-round hash), `POST /api/auth/login` (returns access + refresh JWT pair), `POST /api/auth/refresh` (validates refresh token, issues new pair), `GET /api/auth/me` (returns current user profile from JWT). Ran `prisma validate` + `prisma generate` + `tsc --noEmit` — all clean.
- Files affected: `app/prisma/schema.prisma`, `app/prisma/migrations/0018_users/migration.sql`, `app/src/lib/jwt.ts`, `app/src/lib/auth.ts`, `app/src/app/api/auth/register/route.ts`, `app/src/app/api/auth/login/route.ts`, `app/src/app/api/auth/refresh/route.ts`, `app/src/app/api/auth/me/route.ts`
- Learnings for next cycles:
  - Patterns: JWT secret from `process.env.JWT_SECRET ?? "dev-secret"` — always provide a dev fallback to avoid runtime errors in test environments.
  - Patterns: bcrypt.hash requires async/await — bcryptjs is the CJS-compatible drop-in for Next.js edge/node environments.
  - Patterns: Resolving FK stubs from earlier migrations: use ALTER TABLE ... ADD CONSTRAINT ... FOREIGN KEY ... REFERENCES in the migration that creates the target table.
  - Traps: `UserRole` enum from `@/generated/prisma/client` includes `public` which is a reserved word in PostgreSQL but safe as an enum value when quoted.
